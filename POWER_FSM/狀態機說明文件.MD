Power Supply State Machine (電源順序控制狀態機)

這是一個專為電源轉換器 (Power Converter) 設計的通用狀態機。它解決了電源系統中最核心的「時序控制」問題，確保開關機流程的安全與穩定。

🎯 設計目標

安全性 (Safety)：確保故障 (Fault) 發生時能立即切斷 PWM，且必須手動清除才能重啟。

可移植性 (Portability)：透過 PowerOps 結構體將邏輯與硬體分離 (Decoupling)。FSM 不知道你是用 TI C2000 還是 STM32，它只知道要「開 PWM」或「關 PWM」。

非阻塞軟啟動 (Non-blocking Soft Start)：使用 RunTick 逐步增加參考電壓，而不是使用 delay 卡住 CPU。

🔄 狀態流程圖 (State Diagram)

[PS_IDLE] (待機, PWM Off)
    | <--- (EVT_CMD_OFF) ---+
    | (EVT_CMD_ON)          |
    v                       |
[PS_CHECK_INIT] (檢查 Vin) -+ (檢查失敗)
    | (檢查通過)             |
    v                       |
[PS_SOFT_START] (軟啟動) ----+
    | (VRef ramp up 0% -> 100%)
    v
[PS_RUN] (正常運作 Loop) ----+
    |
    | (任何時刻發生 EVT_FAULT_TRIP)
    v
[PS_FAULT] (故障鎖定, PWM Off)
    |
    +--- (EVT_CLR_FAULT) ---> [PS_IDLE]


🛠 如何整合到 TI C2000 專案？

1. 實作硬體介面 (Main.c)

你需要實作 PowerOps 定義的四個函數。這是唯一會碰到 EPwmRegs 的地方。

void My_EnablePWM(bool enable) {
    if (enable) {
        EALLOW;
        // 清除 Trip Zone，允許 PWM 輸出
        EPwm1Regs.TZCLR.bit.OST = 1; 
        EDIS;
    } else {
        EALLOW;
        // 強制 Trip Zone 動作，關閉 PWM
        EPwm1Regs.TZFRC.bit.OST = 1; 
        EDIS;
    }
}

const PowerOps hardware_ops = {
    .enable_pwm_output = My_EnablePWM,
    // ... 填入其他函數
};


2. 初始化與中斷配置

在 main() 中初始化 FSM，並將 FSM_RunTick 放入你的 ISR (例如 1kHz 的控制迴路或背景迴路)。

// ISR (例如 1ms 觸發一次)
interrupt void cpu_timer0_isr(void) {
    FSM_RunTick(&g_PowerFSM);
    PieCtrlRegs.PIEACK.all = PIEACK_GROUP1;
}


3. 結合 Parser (接收指令)

當 Parser 解析出指令後，直接呼叫 FSM_HandleEvent。

void On_Command_Received(uint8_t cmd) {
    if (cmd == CMD_ON) {
        FSM_HandleEvent(&g_PowerFSM, EVT_CMD_ON);
    }
}


💡 進階技巧

軟啟動曲線：目前的 RunSoftStart 是線性的。你可以修改 power_fsm.c 實作 S-Curve 軟啟動，以減少湧浪電流 (Inrush Current)。

故障碼：fault_code 可以是一個 Bitmask，分別代表 OVP, OCP, OTP，方便回傳給上位機進行診斷。
