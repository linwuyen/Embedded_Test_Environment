Generic Ring Buffer for Embedded Systems (C Module)

這是一個專為嵌入式系統 (Embedded Systems) 設計的通用環形緩衝區 (Ring Buffer) 模組。
特別針對 TI C2000 MCU 及電源控制應用進行了考量，但其 高內聚、低耦合 的特性使其適用於任何 C 語言開發環境 (STM32, PIC, AVR, etc.)。

✨ 特點 (Features)

高內聚 (High Cohesion)：資料結構與操作邏輯封裝完整。

低耦合 (Low Coupling)：不依賴特定硬體標頭檔，移植性極高。

非阻塞 (Non-blocking)：讀寫操作立即返回成功或失敗，適合控制迴路 (Control Loop)。

記憶體安全 (Static Allocation)：不使用 malloc/free，避免記憶體破碎化 (Fragmentation)。

類型可配置 (Configurable Type)：透過 typedef 輕鬆切換儲存資料類型 (如 uint8_t 用於 UART，uint16_t 用於 12-bit ADC)。

📂 檔案結構 (Files)

ring_buffer.h: 標頭檔，定義結構與 API 介面。

ring_buffer.c: 實作檔，包含邏輯處理與防呆機制。

main.c: 使用範例，展示如何在 ISR (中斷) 與 Main Loop (主迴圈) 之間傳遞 UART 與 ADC 資料。

🚀 快速開始 (Quick Start)

1. 引入與定義

在你的專案中引入標頭檔，並定義實體記憶體空間與控制結構：

#include "ring_buffer.h"

// 定義實體記憶體 (不用 malloc)
#define RX_SIZE 64
rb_item_t rx_buffer_mem[RX_SIZE];

// 定義管理結構
RingBuffer rx_rb;


2. 初始化

在系統初始化的階段 (e.g., main()) 呼叫：

RB_Init(&rx_rb, rx_buffer_mem, RX_SIZE);


3. 在中斷 (ISR) 中寫入

適合 UART 接收或 ADC 採樣：

void UART_RX_ISR(void) {
    uint8_t data = Hardware_Read_Register();
    RB_Write(&rx_rb, data); // 若緩衝區滿，會自動丟棄並回傳 false
}


4. 在主迴圈 (Main Loop) 中讀取

適合非即時性的資料處理：

void main_loop(void) {
    rb_item_t data;
    while (RB_Read(&rx_rb, &data)) {
        // 處理資料...
    }
}


⚠️ 注意事項 (Notes)

Concurrency: 此實作在「單一生產者 (ISR) - 單一消費者 (Main)」模型下通常是安全的。但若有多個中斷同時寫入同一個 Buffer，請務必在 RB_Write 前後加上 Critical Section 保護 (如 DINT/EINT)。

Data Type: 若需儲存 16-bit 或 32-bit 數據，請修改 ring_buffer.h 中的 rb_item_t 定義。
