Generic Protocol Parser (通用通訊協定解析器)

這是一個專為嵌入式系統設計的 非阻塞 (Non-blocking) 封包解析模組。
採用 有限狀態機 (FSM) 與 回呼函式 (Callback) 架構，實現「高內聚、低耦合」，特別適合對即時性要求極高的電源控制系統 (Power Control)。

🎯 核心特點

絕對非阻塞：解析過程只用 if/switch，沒有 while 等待迴圈，保證不卡死控制迴路 (Control Loop)。

硬體無關 (Low Coupling)：不知道資料從哪來 (UART/CAN/SPI)，也不管解析完要做什麼，只負責「拆封包」。

物件導向 (High Cohesion)：所有狀態變數封裝在 struct 中，可同時宣告多個 Parser 實體 (例如同時解析 PC 指令與周邊板通訊)。

防止破碎化：使用靜態陣列 (rx_buffer)，不使用 malloc，適合 TI C2000 等資源受限 MCU。

⚙️ 運作原理：有限狀態機 (FSM)

解析器像一個過篩器，一個 Byte 一個 Byte 檢查，狀態流程如下：

[初始狀態]
     |
     v
[等待表頭1 (0x55)] --(非 0x55)--> [丟棄，繼續等待]
     | (收到 0x55)
     v
[等待表頭2 (0xAA)] --(非 0xAA)--> [回到起點]
     | (收到 0xAA)
     v
[等待長度 (Length)] --(長度過大)--> [回到起點]
     | (合法長度 N)
     v
[接收資料 (Payload)] <--(接收中)-- [存入 Buffer]
     | (收滿 N 個 Byte)
     v
[檢查校驗 (Checksum)] --(失敗)--> [回到起點 (丟棄)]
     | (成功!)
     v
✅ [觸發 Callback 函式] --> [回到起點]


🚀 快速開始 (Quick Start)

1. 定義回呼函式 (Callback)

這是解析成功後，「你」想要執行的業務邏輯。

// 當收到完整封包時，Parser 會自動呼叫這個函式
void My_Packet_Handler(uint8_t *payload, uint16_t len) {
    // payload[0] 通常是 Command ID
    switch(payload[0]) {
        case 0x01: // 例如：設定電壓
            // 執行你的電源控制邏輯
            break;
    }
}


2. 初始化

在 main() 初始化的部分：

#include "protocol_parser.h"

ProtocolParser my_parser; // 宣告解析器物件

void main() {
    // 初始化並註冊你的 Callback
    Parser_Init(&my_parser, My_Packet_Handler);
    
    // ... 其他初始化
}


3. 餵入資料 (Input)

你可以直接在中斷 (ISR) 餵資料，或者配合 Ring Buffer 在主迴圈餵資料 (推薦)。

方法 A：配合 Ring Buffer 在主迴圈處理 (最推薦)

while (RB_Read(&uart_rb, &byte)) {
    Parser_InputByte(&my_parser, byte); 
    // 若解析成功，內部會自動跳去執行 My_Packet_Handler
}


方法 B：直接在中斷處理 (僅限簡單邏輯)

void UART_RX_ISR(void) {
    uint8_t byte = UART_Read_Reg();
    Parser_InputByte(&my_parser, byte); 
}


🛠 設定與修改 (Configuration)

打開 protocol_parser.h 的上方定義區進行修改：

// 定義最大封包長度 (Bytes)
#define MAX_PACKET_SIZE  64      

// 定義你的協定表頭 (Header)
#define FRAME_HEADER_1   0x55    
#define FRAME_HEADER_2   0xAA    


若需修改 校驗算法 (Checksum) (例如改為 CRC16)，請修改 protocol_parser.c 中的 CalculateChecksum 函式。

⚠️ TI C2000 開發者注意事項

資料型態：程式碼使用標準 uint8_t。在 C2000 上，char 是 16-bit，但 uint8_t 依然只取低 8 位，邏輯是通用的。注意與 PC 端通訊時的 Byte Alignment。

即時性：雖然此 Parser 是非阻塞的，但若 Callback 裡的邏輯太複雜 (例如寫 Flash)，還是會卡住。建議 Callback 只做簡單的 Flag 設定或數值更新。
